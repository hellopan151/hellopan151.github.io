差異比對

使用 git diff 來取得目錄之間的差異

  1. git diff 預設以目前的工作目錄為主，其次以目前的所在的 checkout 為次
  2. git diff -> 工作目錄 vs Stage ( 已經在 stage 追縱的檔案 )
  3. git diff HEAD -> 工作目錄 vs HEAD
  4. git diff –cached -> 工作目錄 vs Stage
  5. git diff –cached HEAD -> Stage vs HEAD
  • 想在 git add 之前看看到底改了那一些資料，當你更改了工作目錄的一些檔案 ( 那些檔案是已經追縱的 tracked ) ，

git diff PS. 等同 git status ( 暫時不管 untracked 的資訊 )

  • 想知道這次 commit 時，會更動那些資料，想在 git commit 之前看到改了那些 ( stage vs commit )

git diff –cached

想知道這次 commit 時，會更動那些資料，想在 git commit 之前看到改了那些 ( 工作目錄 vs commit )

git diff HEAD

  • 想知道這次的 commit 與前一次的 commit 差異

git diff HEAD^ HEAD
