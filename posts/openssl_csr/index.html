每年都要申請多個網站憑證，憑記憶輸入資料總是不確實，終於找到一行命令列就直接產出 CSR / Key 檔案的方式，每年只要執行一次就可以批次將檔案一口氣全部產出，方便又不容易出錯。

SSL CSR 產生方式

純命令列方式 ( 現行使用方式 )

在 linux / bash 之中直接使用以下指令，產生 key 檔案以及 csr 檔案後，csr 檔案交給代理商即可完成大部份事情。

# star ssl ( domain ssl )
openssl req -new -newkey rsa:2048 -nodes -keyout star.domain.com.tw.key -out star.domain.com.tw.csr -subj "/C=TW/ST=TAIWAN/L=TAIPEI/O=公司英文名/OU=部門英文名/CN=*.domain.com.tw"
# www ssl ( single web site )
openssl req -new -newkey rsa:2048 -nodes -keyout www.domain.com.tw.key -out www.domain.com.tw.csr -subj "/C=TW/ST=TAIWAN/L=TAIPEI/O=公司英文名/OU=部門英文名/CN=www.domain.com.tw"


記得將網址以及合適的資訊放在 subject 的區段。


reference:

  1. OpenSSL Quick Reference Guide
  2. How to create a certificate using OpenSSL with Subject Alternative Name field (SAN)
  3. 很棒的一本參考書︰ Openssl Cookbook


利用預先寫好的 config 方式 ( 可參考 )

openssl req -new -nodes -sha256 -utf8 -newkey rsa:2048 -keyout www.domain.com.tw.key -out www.domain.com.tw.csr -config domain.conf


domain.conf 檔案內容如下

[req]
prompt = no
default_md = sha256
default_bits = 2048
distinguished_name = dn
x509_extensions = v3_req

[dn]
C = TW
ST = TAIWAN
L = TAIPEI
O = 公司英文名
OU = 部門英文名
CN = www.domain.com.tw

[v3_req]


reference:

  1. OpenSSL 操作筆記 - 產生 CSR
  2. 保哥的筆記 - 如何使用 OpenSSL 工具產製 CSR 憑證要求檔並成功申請正式 TLS 憑證

在 windows IIS 的傳統 Microsoft 方式 ( 手工點選操作界面，已棄用 )

  1. 先在 IIS 提出 CSR ，將 CSR 傳給憑證代理商
  2. 代理商以電話、電子郵件或是 DNS 的方式向公司做確認
  3. 代理商將憑證傳回
  4. 我方將它轉成 pfx 格式，並滙回原來的 IIS 網站



reference:
